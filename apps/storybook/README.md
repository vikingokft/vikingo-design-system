# @vikingo/storybook

Storybook 8 instance for the Vikingo Design System. The deployed site is the **canonical reference** for component behavior, props, and visual variations.

> **Live:** https://vikingo-storybook.vercel.app

## Run locally

```bash
# from repo root
pnpm storybook
```

Opens at http://localhost:6006. Hot-reloads on changes to either `apps/storybook/stories/` or `packages/ui/src/`.

## Build static site

```bash
pnpm --filter @vikingo/storybook run build-storybook
```

Output: `apps/storybook/storybook-static/`. Vercel deploys from this.

## Story conventions

- Files: `apps/storybook/stories/{primitives,layout,charts,tokens}/<Name>.stories.tsx`
- Filename matches the component class — `Button.stories.tsx`, `DataTable.stories.tsx`.
- Stories import from `@vikingo/ui` (the published API), never from internal paths. This way, breaking the export map breaks Storybook — a useful guardrail.
- Story title format: bilingual `'English · Magyar'`.
- `tags: ['autodocs']` always present so the Docs tab is generated.
- `layout: 'centered'` for most components, `'padded'` for wider ones (DataTable, PageLayout, Sidebar).
- Use `render:` for stories that need state, `args:` for static prop variations.

See [CONTRIBUTING.md](../../CONTRIBUTING.md#story-template) for the canonical story template.

## Theme toggle

The toolbar has a light/dark switch (`@storybook/addon-themes`) that toggles the `dark` class on `<html>`. This exercises the same dark-mode path consumers use — no separate wiring.

## Reduced-motion toggle

The toolbar also exposes a global `force-reduce-motion` toggle that adds the `force-reduce-motion` class on `<body>`, simulating a user with `prefers-reduced-motion: reduce` set in their OS. Useful for verifying that animations gracefully short-circuit.

## Adding a new story

1. Create `apps/storybook/stories/primitives/<Name>.stories.tsx` (or the matching subfolder for layout/charts/tokens).
2. Copy the story template from CONTRIBUTING.md.
3. Add at least: `Default`, `WithIcon` (if the component supports icons), `Disabled` (if applicable).
4. Confirm both light and dark mode look correct.
5. Confirm the autogenerated Docs tab shows props with descriptions.

## Vite alias setup

`vite.config.ts` aliases `@vikingo/ui` to the TypeScript source so we develop against `src/` directly — no rebuild loop. Also aliases the legacy `@vikingo/ui/styles` and `@vikingo/ui/fonts` paths for backwards-compatible story examples.
